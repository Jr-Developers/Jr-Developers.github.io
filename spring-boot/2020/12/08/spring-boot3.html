Spring Data JPA Multi DataSource

by 주니어 개발자

December 8, 2020
in Spring-boot

DB 서버를 두개를 사용해야 하거나 두 개의 스키마를 사용해야 하는 경우 두 개의 DataSource를 구성해야 한다.

두 개의 DataSource 설정을 위해 Spring Boot Reference Doc#Configure Two DataSources에서 제시 하는 설정을 참고 하였으며 이에 따른 두개의 EntityManager설정은 Spring Boot Reference Doc#Use Two EntityManagers과 Spring Data JPA ReferenceDoc#Annotation-based Configuration를 참고하였다.

정상적으로 Multi Source 가 동작하는지 확인하기 위해서 두 개의 Spring Boot서버에서 두 개의 H2을 기동시키며 자신의 서버 H2 와 나머지 서버의 H2 서버에 접속해 볼 것이다.

첫 번째 서버는 Member서버 라고 지징하겠으며 MEMBER라는 Table 하나를 갖는다. 해당 서버에서는 자신의 H2 DataSource 만 이용하며 Rest API를 통해 CLUB의 정보를 조회 할 것이다. 두 번째 서버는 Club서버 라고 지칭하겠으며 CLUB이라는 Table 하나를 갖는다. 해당 서버에서는 두 개의 H2 DataSource 를 이용하여 데이터를 조회 할 것 이다.

Member서버에는 Club서버에서 Member서버의 H2에 접속하기 위해 이전의 포스팅한 H2 TCP 포트 개방 설정을 하였다.

  • Member서버 - Main

Member서버 자신의 MEMBER 테이블 조회와 RestTemplate를 이용한 Club서버의 CLUB 테이블 조회로 구성하였다.

Club서버의 application.yml에는 두 개의 DataSource 설정을 위한 자신의 H2 서버와 Member서버의 H2 서버의 정보를 포함하였으며 Hikari initialization-fail-timeout 설정을 통해 Club서버가 먼저 기동될 경우 Member서버의 기동을 잠시 기다리도록 설정 하였다.
추가적으로 Hikari Connection Pool 설정은 HikariCP Dead lock에서 벗어나기 (이론편)과 HikariCP Dead lock에서 벗어나기 (실전편)에 자세한 실전 경험과 내용이 정리되어 있다.

  • Member서버 - application.yml

두 개의 DataSource를 구성하기 여러 방법이 있으며 나는 Spring Boot Reference Doc에서 제시한 방법을 이용하였다. 해당 방법은 Spring 내부에서 기본적으로 사용하는 DataSource 설정 방법인것으로 확인하였다.

  • Spring 기본 DataSource 설정 - DataSourceConfiguration.java
  • Club서버 - clubdb DataSource 설정

@EnableJpaRepositories를 통해 해당 DataSource에서 사용될 JPA Repostiroy의 Package의 경로와 EntityManagerFactory, TransactionManager를 등록하며 EntityManagerFactory 생성시 Hibernate 설정을 추가 할 수 있으며 기본으로 사용될 DataSource가 아닌 경우 PersistenceUnitName를 설정하여 EntityManager를 가져 올 수 있다.
